# p2p-chat — локальный P2P мессенджер со сквозным шифрованием

CLI/TUI мессенджер для общения в локальной сети (LAN) без центрального
сервера. Узлы находят друг друга автоматически (mDNS) или по вручную
введённому адресу, и обмениваются сквозь-зашифрованными сообщениями.

## Зафиксированные требования

| Тема | Решение |
|------|---------|
| Сеть | LAN + ручной ввод адреса (`--dial`) |
| Язык | Rust (edition 2024) |
| Формат чата | MVP — 1-на-1; архитектура заложена под группы |
| Интерфейс | TUI (ratatui), тема «неоновый циан»; конфиг вводится на стартовом экране |
| Запуск | Один бинарь `l0kus-messanger`; порт/ник/адрес — на стартовом экране TUI, флаги CLI — оверрайды |
| Крипто 1-на-1 | Olm = X3DH-подобный handshake + Double Ratchet (forward secrecy + post-compromise security) |
| Идентичность | Долговременный Ed25519-ключ (vodozemac Account), ID = fingerprint, TOFU + ручная сверка |
| История сообщений | Эфемерная — только в памяти, при выходе исчезает |
| Доп. функции | Автообнаружение (mDNS), передача файлов, присутствие/статус |
| Сетевой фундамент | rust-libp2p |
| Транспорт | QUIC (основной) + TCP (запасной) |

## Архитектура

- **libp2p** — транспорт (QUIC/TCP), обнаружение (mDNS), идентификация
  (identify), keepalive (ping) и протокол `request-response` (`/p2p-chat/1`)
  для доставки сообщений. libp2p PeerId эфемерный (новый на запуск).
- **vodozemac (Olm)** — сквозное шифрование поверх libp2p-потока. Это источник
  истины о безопасности контента: forward secrecy + post-compromise security.
  Megolm из той же библиотеки закроет будущие группы (sender keys).
- **Идентичность** = Ed25519-ключ из `vodozemac::Account`, персистится на диск
  (`$XDG_DATA_HOME/p2p-chat/identity.bin`, права 0600), поэтому fingerprint
  стабилен между запусками. Доверие к пирам — TOFU в памяти + ручная сверка
  fingerprint в TUI.

### Поток установления сессии (без гонок)

1. Соединение (QUIC/TCP) → `identify` обменивается PeerId/адресами.
2. Обе стороны шлют `KeyBundle` (ed25519 + curve25519 + one-time key + ник).
3. Инициатор = пир с **меньшим PeerId**. Получив бандл, создаёт outbound
   Olm-сессию и шлёт зашифрованный `Hello` (PreKey-сообщение).
4. Ответчик на PreKey создаёт inbound-сессию. Дальше одна общая сессия,
   обычные (Normal) сообщения в обе стороны.

Полезная нагрузка внутри Olm — enum `Plain` (`Hello`, `Text`, файловые чанки),
сериализуется через `rmp-serde`.

### Старт узла (отложенный листенер)

Листенер не поднимается на старте процесса: `Engine::run` крутит цикл, а
слушание/набор адресов запускается по команде `UiCommand::Start { nick, port,
dials }` (идемпотентна, флаг `Engine.started`). Это позволяет вводить порт/ник
уже в TUI. Источник `Start`:
- **интерактивный TUI** (`./l0kus-messanger` без флагов) — команду шлёт
  стартовый экран после подтверждения;
- **неинтерактивно** (любой из `--port/--nick/--dial` или `--headless`) — `main`
  шлёт `Start` сразу, экран настройки пропускается.
`Config::is_interactive()` решает, какой путь выбрать.

## Структура модулей

```
src/
  main.rs          точка входа, парсинг CLI, ветвление запуска, мост каналов
  config.rs        CLI-аргументы (оверрайды); is_interactive(), default_nick()
  settings.rs      config.toml в каталоге данных (ник/порт/адрес «запоминать»)
  identity.rs      vodozemac Account (persist), fingerprint, set_nick
  message.rs       доменные типы; enum UiCommand (вкл. Start) / AppEvent
  crypto.rs        Olm-сессии на пира: encrypt/decrypt, Plain payload
  net/
    mod.rs         re-export
    behaviour.rs   композиция NetworkBehaviour (mdns/identify/ping/request-response)
    protocol.rs    wire-типы (WireMsg / Ack)
    swarm.rs       сборка Swarm + цикл событий, мост с engine
  engine.rs        оркестратор: net-события <-> crypto <-> UI-канал; start()
  ui/
    mod.rs         цикл TUI, StartupConfig
    app.rs         состояние: Screen, SetupForm, пиры, чат (+ время), скролл, help
    theme.rs       палитра «неоновый циан» и хелперы блоков
    view.rs        рендер ratatui (чат, стартовый экран, оверлей помощи)
    events.rs      обработка клавиатуры (маршрутизация Setup/Chat)
  file_transfer.rs чанкинг + сборка файлов
```

## Запуск и проверка

```bash
cargo build
cargo clippy
cargo test            # round-trip Olm, сериализация wire-типов

make                  # release + симлинк ./l0kus-messanger в корне

# Голый запуск — стартовый экран TUI (ник/порт/адрес вводятся там):
./l0kus-messanger

# Два узла в одной LAN — найдут друг друга по mDNS (ник укажите на экране).

# На одной машине — нужны РАЗНЫЕ каталоги данных (иначе общий identity.bin
# и одинаковый fingerprint). Флаги пропускают стартовый экран:
./l0kus-messanger --data-dir /tmp/alice --port 4001 --nick alice
./l0kus-messanger --data-dir /tmp/bob   --port 4002 --nick bob \
    --dial /ip4/127.0.0.1/udp/4001/quic-v1
```

> Идентичность одна на каталог данных. На разных машинах флаг не нужен —
> у каждой свой `$XDG_DATA_HOME/p2p-chat`. Эквивалент флага — переменная
> окружения `P2P_CHAT_DATA_DIR`.

### Безголовый режим (CLI для скриптов и тестов)

`--headless` запускает узел без TUI: команды читаются построчно из stdin,
события печатаются в stdout (`READY/LISTEN/PEER/PRESENCE/MSG/FILE/STATUS`).
Команды stdin: обычный текст → отправка первому пиру, `/dial <addr>`,
`/file <путь>`, `/verify`, `/quit`. Несколько узлов на одной машине —
через `P2P_CHAT_DATA_DIR=<каталог>` (своя идентичность на узел).

E2E-проверка двух узлов по QUIC (текст в обе стороны + файл) выполняется
именно через этот режим; покрывает handshake, шифрование, присутствие,
автообнаружение и передачу файлов.

Логи пишутся в файл (`$XDG_DATA_HOME/p2p-chat/p2p-chat.log`), а не в TUI,
управляются `RUST_LOG`.

## TUI: горячие клавиши

Стартовый экран (Setup): `Tab`/`↑`/`↓` — поле, `Space` — галка «запоминать»,
`Enter` — запустить узел, `Esc` — выход.

Чат:
- `Tab` / `↑`/`↓` — переключение между пирами в списке
- `Enter` — отправить сообщение из строки ввода
- `PgUp` / `PgDn` — прокрутка истории чата
- `Ctrl+V` — пометить выбранного пира как проверенного (сверка fingerprint)
- `/file <путь>` — отправить файл выбранному пиру (вводится в строке ввода)
- `/dial <адрес>` — подключиться к пиру вручную
- `?` / `F1` — оверлей со справкой по клавишам
- `Esc` / `Ctrl+C` — выход
