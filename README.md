# p2p-chat

Локальный P2P мессенджер со сквозным шифрованием и TUI. Без серверов и
аккаунтов: узлы находят друг друга в локальной сети и обмениваются
зашифрованными сообщениями и файлами напрямую.

- 🔒 **Сквозное шифрование** — Olm (X3DH + Double Ratchet) поверх libp2p:
  forward secrecy и post-compromise security.
- 🛰 **Автообнаружение** в LAN по mDNS + ручное подключение по адресу.
- 🖥 **TUI** в стиле Irssi/WeeChat (список пиров, чат, ввод).
- 📎 **Передача файлов** по тому же зашифрованному каналу.
- 👻 **Эфемерность** — история сообщений живёт только в памяти сессии.
- 🪪 **Долговременная идентичность** — стабильный Ed25519-fingerprint для
  ручной сверки собеседника (защита от man-in-the-middle).

> ⚠️ Учебный проект. Криптопримитивы взяты из проаудированной библиотеки
> [vodozemac](https://github.com/matrix-org/vodozemac), но протокол поверх неё
> аудита не проходил — не полагайтесь на него в задачах с реальными рисками.

## Установка

Нужен Rust (edition 2024, проверено на 1.95).

```bash
make            # собирает release и кладёт симлинк ./l0kus-messanger в корень
# либо вручную:
cargo build --release   # бинарь: target/release/l0kus-messanger
```

## Быстрый старт

Запуск — одним файлом:

```bash
./l0kus-messanger
```

Откроется **стартовый экран** TUI: введите ник и (необязательно) порт и адрес
для подключения, при желании поставьте галку «Запоминать настройки» и нажмите
`Enter`. Узел начнёт слушать сеть и появится в чате. Поля переключаются `Tab`,
галка — `Space`, выход — `Esc`.

> Флаги командной строки (см. ниже) — это **оверрайды**: если передать любой из
> `--port/--nick/--dial` (или `--headless`), стартовый экран пропускается и узел
> запускается сразу. Удобно для скриптов и тестов.

### Две машины в одной локальной сети

Они найдут друг друга автоматически по mDNS — достаточно запустить
`./l0kus-messanger` на обеих и указать ники на стартовом экране.

### Два узла на одной машине

Нужны **разные каталоги данных** (иначе общая идентичность и одинаковый
fingerprint). Здесь удобнее флаги-оверрайды:

```bash
./l0kus-messanger --data-dir /tmp/alice --port 4001 --nick alice
./l0kus-messanger --data-dir /tmp/bob   --port 4002 --nick bob \
    --dial /ip4/127.0.0.1/udp/4001/quic-v1
```

## Как пользоваться (TUI)

| Действие | Клавиша / команда |
|---|---|
| Переключить активного пира | `Tab` или `↑` / `↓` |
| Отправить сообщение | ввести текст → `Enter` |
| Прокрутка истории чата | `PgUp` / `PgDn` |
| Сверить пира (доверие) | `Ctrl+V` |
| Подключиться вручную | `/dial <адрес>` |
| Выбрать файл (браузер) | `Ctrl+F` |
| Отправить файл по пути | `/file <путь>` |
| Справка по клавишам | `?` или `F1` |
| Выход | `Esc` или `Ctrl+C` |

Обозначения в чате: `→` — ваши сообщения, `←` — входящие, `·` — системные
(файлы, статусы), слева — время `[ЧЧ:ММ]`. Сверху панели — ваш ник и
fingerprint, снизу — строка статуса с подсказками и адресом для подключения.

### Поиск собеседников

1. **Автоматически (mDNS).** В одной LAN ничего делать не нужно — пиры
   появляются в списке сами.
2. **Вручную.** Если автообнаружение недоступно (разные подсети, VPN, проброс
   портов), наберите:
   ```
   /dial /ip4/192.168.1.50/udp/4001/quic-v1
   ```
   Поддерживается и короткая форма `/dial 192.168.1.50:4001` (трактуется как
   QUIC). Адрес для подключения собеседник видит в строке статуса при старте.

### Что значит «не сверен» и зачем это нужно

Сообщения шифруются с самого начала — но программа не может *сама* убедиться,
что ключ собеседника принадлежит именно ему, а не перехватчику в сети
(man-in-the-middle). Поэтому каждый новый пир помечается `[не сверен]`.

**Fingerprint** — это отпечаток долговременного ключа собеседника. Чтобы
подтвердить личность:

1. Сравните fingerprint **по другому каналу** (голосом, лично, в другом
   мессенджере). У вас он показан вверху своей панели, fingerprint собеседника —
   рядом с его ником. Они должны совпасть у обоих с тем, что показывает другая
   сторона.
2. Если совпало — выберите пира и нажмите **`Ctrl+V`**. Метка станет
   `[доверен ✓]`.

Доверие эфемерно (как и история) — сверка повторяется в новой сессии.

## Передача файлов

Два способа выбрать файл активному пиру:

- **Браузер** — нажмите `Ctrl+F`. Откроется навигация по каталогам с подсветкой
  как в терминале: директории — синим (`/`), исполняемые — зелёным (`*`),
  ссылки — голубым (`@`). `↑`/`↓` — выбор, `Enter` — войти в каталог или
  отправить файл, `Backspace` — на уровень выше, `Esc` — отмена.
- **По пути** — команда в строке ввода:
  ```
  /file ~/photos/cat.jpg
  ```

Файл шифруется и отправляется активному пиру. Принятые файлы сохраняются в
каталог загрузок: `<data-dir>/downloads/` (по умолчанию
`~/.local/share/p2p-chat/downloads/`).

## Аргументы командной строки

Все флаги необязательны и работают как **оверрайды** стартового экрана: передача
любого из `--port/--nick/--dial` (или `--headless`) пропускает экран настройки.

| Флаг | Описание |
|---|---|
| `--port <PORT>` | Порт для входящих соединений (QUIC и TCP). `0` — случайный. |
| `--nick <NAME>` | Отображаемый ник (по умолчанию — имя пользователя ОС). |
| `--dial <ADDR>` | Подключиться к адресу (multiaddr или `ip:port`). Можно несколько раз. |
| `--data-dir <DIR>` | Каталог данных (идентичность, логи, загрузки). |
| `--headless` | Без TUI: команды из stdin, события в stdout (скрипты/тесты). |

Каталог данных по умолчанию — `$XDG_DATA_HOME/p2p-chat`; переопределяется
флагом `--data-dir` или переменной `P2P_CHAT_DATA_DIR`. Логи пишутся в
`<data-dir>/p2p-chat.log` (уровень — через `RUST_LOG`).

Галка «Запоминать настройки» на стартовом экране сохраняет ник/порт/адрес в
`<data-dir>/config.toml` и предзаполняет ими форму при следующем запуске.

## Безголовый режим

Для скриптов и автоматизации (TUI требует терминал):

```bash
./l0kus-messanger --headless --port 4001 --nick alice
```

stdin построчно: текст → отправка первому пиру; `/dial <addr>`, `/file <путь>`,
`/verify`, `/quit`. stdout — машиночитаемые события:
`READY / LISTEN / PEER / PRESENCE / MSG / FILE / STATUS`.

## Как это устроено

```
TUI (ratatui) ──каналы──> Engine ──> libp2p Swarm (QUIC/TCP, mDNS, request-response)
                            │
                            └──> Olm-сессии (vodozemac): шифрование/расшифровка
```

- **Транспорт:** libp2p, QUIC (основной) + TCP (запасной). Обнаружение — mDNS.
- **E2E:** при соединении стороны обмениваются ключевыми бандлами; инициатор
  (меньший PeerId) устанавливает Olm-сессию первым PreKey-сообщением. Дальше —
  единая Double Ratchet сессия в обе стороны.
- **Идентичность:** Ed25519-ключ vodozemac, персистится на диск
  (`identity.bin`, права `0600`), fingerprint стабилен между запусками.

Подробности и архитектурные решения — в [`CLAUDE.md`](./CLAUDE.md).

## Безопасность: что есть и чего нет

- ✅ Сквозное шифрование, forward secrecy, post-compromise security (Olm).
- ✅ Сверка собеседника по fingerprint (TOFU + ручное подтверждение).
- ❌ Группы пока не реализованы (архитектура заложена под Megolm).
- ❌ Идентичность на диске не зашифрована паролем.
- ❌ Метаданные (кто с кем на связи в LAN) не скрываются.
