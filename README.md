# p2p-chat

Локальный P2P-мессенджер со сквозным шифрованием и текстовым интерфейсом.
Без серверов и аккаунтов: узлы находят друг друга в локальной сети и
обмениваются зашифрованными сообщениями и файлами напрямую.

- Сквозное шифрование Olm (X3DH + Double Ratchet) поверх libp2p: forward
  secrecy и post-compromise security.
- Автообнаружение в LAN по mDNS и ручное подключение по адресу.
- TUI со списком пиров, чатом и строкой ввода.
- Передача файлов по тому же зашифрованному каналу.
- Эфемерность: история сообщений живёт только в памяти сессии.
- Долговременная идентичность: стабильный Ed25519-fingerprint для ручной
  сверки собеседника (защита от man-in-the-middle).

Учебный проект. Криптопримитивы взяты из проаудированной библиотеки
[vodozemac](https://github.com/matrix-org/vodozemac), но протокол поверх неё
аудита не проходил — не полагайтесь на него в задачах с реальными рисками.

## Установка

В корне репозитория лежит собранный бинарь для Linux x86-64 — его можно
запускать сразу:

```bash
./l0kus-messanger
```

Сборка из исходников (нужен Rust, edition 2024):

```bash
make                    # release + симлинк ./l0kus-messanger в корне
cargo build --release   # либо вручную: target/release/l0kus-messanger
```

## Быстрый старт

```bash
./l0kus-messanger
```

Откроется стартовый экран: введите ник и, по желанию, порт и адрес для
подключения, при необходимости включите «Запоминать настройки» и нажмите
`Enter`. Поля переключаются `Tab`, галка — `Space`, выход — `Esc`.

Флаги командной строки — это оверрайды: передача любого из
`--port/--nick/--dial` (или `--headless`) пропускает стартовый экран и
запускает узел сразу.

### Две машины в одной сети

Запустите `./l0kus-messanger` на обеих и укажите ники — узлы найдут друг друга
по mDNS автоматически.

### Два узла на одной машине

Нужны разные каталоги данных (иначе общая идентичность и одинаковый
fingerprint):

```bash
./l0kus-messanger --data-dir /tmp/alice --port 4001 --nick alice
./l0kus-messanger --data-dir /tmp/bob   --port 4002 --nick bob \
    --dial /ip4/127.0.0.1/udp/4001/quic-v1
```

## Управление (TUI)

| Действие | Клавиша / команда |
|---|---|
| Переключить активного пира | `Tab` или `↑` / `↓` |
| Отправить сообщение | ввести текст → `Enter` |
| Прокрутка истории | `PgUp` / `PgDn` |
| Сверить пира (доверие) | `Ctrl+V` |
| Подключиться вручную | `/dial <адрес>` |
| Пересканировать LAN | `Ctrl+R` или `/refresh` |
| Выбрать файл (браузер) | `Ctrl+F` |
| Отправить файл по пути | `/file <путь>` |
| Справка | `?` или `F1` |
| Выход | `Esc` или `Ctrl+C` |

В чате: `→` — ваши сообщения, `←` — входящие, `·` — системные, слева — время
`[ЧЧ:ММ]`. Сверху — ваш ник и fingerprint, снизу — строка статуса с адресом
для подключения.

### Поиск собеседников

1. Автоматически по mDNS — в одной LAN пиры появляются в списке сами.
2. Если автообнаружение не сработало (пир запустился позже, разные подсети,
   VPN), нажмите `Ctrl+R` или введите `/refresh` для повторного сканирования.
3. Вручную — `/dial /ip4/192.168.1.50/udp/4001/quic-v1`. Поддерживается
   короткая форма `/dial 192.168.1.50:4001` (трактуется как QUIC). Свой адрес
   собеседник видит в строке статуса.

### Сверка собеседника

Сообщения шифруются с самого начала, но программа не может сама убедиться, что
ключ принадлежит именно собеседнику, а не перехватчику. Поэтому каждый новый
пир помечается `[не сверен]`.

Fingerprint — отпечаток долговременного ключа. Чтобы подтвердить личность,
сравните fingerprint по другому каналу (голосом, лично, в другом мессенджере) и,
если совпало, выберите пира и нажмите `Ctrl+V` — метка станет `[доверен]`.
Доверие эфемерно: сверка повторяется в новой сессии.

## Передача файлов

- Браузер — `Ctrl+F`: навигация по каталогам, `↑`/`↓` — выбор, `Enter` — войти
  или отправить, `Backspace` — выше, `Esc` — отмена.
- По пути — `/file ~/photos/cat.jpg`.

Принятые файлы сохраняются в `<data-dir>/downloads/` (по умолчанию
`~/.local/share/p2p-chat/downloads/`).

## Аргументы командной строки

| Флаг | Описание |
|---|---|
| `--port <PORT>` | Порт входящих соединений (QUIC и TCP). `0` — случайный. |
| `--nick <NAME>` | Отображаемый ник (по умолчанию — имя пользователя ОС). |
| `--dial <ADDR>` | Подключиться к адресу (multiaddr или `ip:port`). Можно несколько раз. |
| `--data-dir <DIR>` | Каталог данных (идентичность, логи, загрузки). |
| `--headless` | Без TUI: команды из stdin, события в stdout. |

Каталог данных по умолчанию — `$XDG_DATA_HOME/p2p-chat`; переопределяется
`--data-dir` или переменной `P2P_CHAT_DATA_DIR`. Логи — в
`<data-dir>/p2p-chat.log` (уровень через `RUST_LOG`).

## Безголовый режим

```bash
./l0kus-messanger --headless --port 4001 --nick alice
```

stdin построчно: текст → отправка первому пиру; `/dial <addr>`, `/file <путь>`,
`/verify`, `/refresh`, `/quit`. stdout — машиночитаемые события:
`READY / LISTEN / PEER / PRESENCE / MSG / FILE / STATUS`.

## Архитектура

```
TUI (ratatui) ──каналы──> Engine ──> libp2p Swarm (QUIC/TCP, mDNS, request-response)
                            │
                            └──> Olm-сессии (vodozemac): шифрование/расшифровка
```

- Транспорт: libp2p, QUIC (основной) + TCP (запасной). Обнаружение — mDNS.
- E2E: при соединении стороны обмениваются ключевыми бандлами; инициатор
  (меньший PeerId) устанавливает Olm-сессию первым PreKey-сообщением, дальше —
  единая Double Ratchet сессия в обе стороны.
- Идентичность: Ed25519-ключ vodozemac, персистится на диск (`identity.bin`,
  права `0600`), fingerprint стабилен между запусками.

## Безопасность

- Есть: сквозное шифрование, forward secrecy, post-compromise security,
  сверка по fingerprint (TOFU + ручное подтверждение).
- Нет: групп (архитектура заложена под Megolm), шифрования идентичности на
  диске паролем, сокрытия метаданных (кто с кем на связи в LAN).
